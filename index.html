Week 1–2 · Urgent
Fill the Critical Gaps
🔴 Must Complete First ~14 hours
OOP Pillars
Critical
  • Encapsulation — private/protected, getters/setters
  • Inheritance — extends, parent::, method overriding
  • Polymorphism — same method, different behavior
  • Abstraction — abstract classes
  • Interfaces — contracts with no implementation
  • Abstract class vs Interface — when to use each
  • Constructor property promotion (PHP 8)
  • Static vs instance methods
SOLID Principles
Critical
  • S — Single Responsibility Principle
  • O — Open/Closed Principle
  • L — Liskov Substitution Principle
  • I — Interface Segregation Principle
  • D — Dependency Inversion Principle
  • Identify SOLID violations in real code
  • Refactor a fat controller using SOLID
Dependency Injection & IoC
Critical
  • What is DI and why it matters
  • Constructor injection (you already do this)
  • Method injection
  • Service Container — what it is, how it resolves
  • Service Providers — boot() vs register()
  • Binding interfaces to implementations
  • Singletons in the container
N+1 Problem — Full Mastery
High
  • Why N+1 kills performance at scale
  • Eager loading — with()
  • Nested eager loading — with('rel.nested')
  • withCount() — count without loading
  • load() — lazy eager loading after fetch
  • Laravel Debugbar — detect N+1 in dev
  • select() — avoid loading unused columns
CSRF & XSS
High
  • What CSRF is and how the attack works
  • How @csrf token protects forms
  • Why Sanctum API tokens aren't vulnerable
  • XSS — injecting malicious JS
  • How Blade {{ }} auto-escapes
  • When to use {!! !!} and why it's dangerous
SQL Injection
High
  • How SQL injection attacks work
  • PDO prepared statements explained
  • Why Eloquent / QB are safe by default
  • Dangerous patterns — raw DB::select with $var
  • DB::raw() — when safe and when not
  • Mass assignment — $fillable vs $guarded

🔵 Deep Knowledge ~21 hours

Critical

  • LEFT JOIN — all from left, null for missing right
  • RIGHT JOIN — all from right
  • CROSS JOIN — cartesian product
  • Self JOIN — joining a table with itself
  • Multiple joins in one query
  • Joins in Eloquent vs Query Builder
  • join() vs leftJoin() in Laravel
Indexing — Deep Mastery
Critical
  • How B-Tree indexes work internally
  • Single-column indexes
  • Composite indexes — column order matters
  • Unique indexes
  • Full-text indexes for search
  • EXPLAIN — read query execution plan
  • Indexes in Laravel migrations
  • When indexes HURT performance (over-indexing)
  • Covering indexes
Transactions & Data Integrity
Critical
  • ACID properties — Atomicity, Consistency, Isolation, Durability
  • DB::transaction() — auto rollback on exception

  • Nested transactions — savepoints

Week 10–13 · Professional
Security, Testing & Clean Code
🟢 Professional Skills ~28 hours
Security — All Major Attacks
Critical
  • SQL Injection — prepared statements (revisit deep)
  • XSS — stored vs reflected vs DOM-based
  • CSRF — tokens, SameSite cookies
  • Mass Assignment — $fillable, $guarded
  • Broken Authentication — token storage best practices
  • Session fixation & hijacking prevention
  • Insecure Direct Object Reference (IDOR)
  • Rate limiting — prevent brute force
  • Environment variables — never expose .env
  • HTTPS and secure cookie flags
Testing — PHPUnit & Pest
Critical
  • Unit tests vs Feature tests — difference
  • php artisan make:test
  • RefreshDatabase trait
  • actingAs($user) — authenticated tests
  • Model Factories — User::factory()->create()
  • assertStatus(), assertJson(), assertJsonStructure()
  • assertDatabaseHas(), assertDatabaseMissing()
  • Mocking — mock services in tests
  • Testing queued jobs
  • Testing middleware
  • Testing file uploads
  • Goal: write 10 feature tests for your API project
Clean Code Practices
Critical
  • Thin controllers — no business logic
  • Service classes — where logic lives
  • Action classes — single-purpose (CreateInvoice)
  • Early returns instead of deep nesting
  • Meaningful variable and method names
  • Avoid magic numbers and magic strings
  • DRY — Don't Repeat Yourself
  • Comments: why, not what
  • PHP 8 features — match, named args, nullsafe
Error Handling & Logging
High
  • Exception handling — try/catch properly
  • Custom exceptions — extend Exception
  • Handler.php — global exception handling
  • render() method for API error responses
  • Logging channels — you already use this, go deeper
  • Log::error(), Log::info(), Log::channel()
  • Sentry / Bugsnag for production error tracking
Git — Professional Level
High
  • Conventional Commits — feat:, fix:, refactor:
  • Branch naming — feature/, fix/, refactor/
  • git rebase vs git merge — difference & when
  • git stash — save work in progress
  • Interactive rebase — squash commits
  • git bisect — find the commit that broke something
  • Writing a proper README.md
  • .gitignore best practices
  • Protecting main branch with rules
API Documentation
Medium
  • Postman collections — document every endpoint
  • Scribe package — auto-generate API docs
  • Swagger / OpenAPI basics
  • Writing good endpoint descriptions
  • Request/response examples in documentation
